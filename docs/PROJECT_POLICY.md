# Project Policy

Project Forge is allowed to create and improve local-first, authorized, and
defensive tools.

It should not create projects that:

- bypass third-party CAPTCHA, rate limits, paywalls, or access controls;
- automate abuse, spam, scraping, credential attacks, or stealth evasion;
- require private secrets to be committed;
- pretend that placeholder code is a finished product.

Every project should include:

- a clear audience;
- a real pain point;
- a small MVP;
- a maintenance-risk note;
- next tasks that are executable.
